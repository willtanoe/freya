### TAHAP 1: BACKEND - ENDPOINTS BARU ✅
--------------------------------------------------------------------------------
FILE: `src/freya/server/routes.py`

TAMBAHAN ENDPOINTS:

1. GET /v1/models/available
   - Return ONLY models yang configured (berdasarkan API keys yang ada)
   - Response: { providers: [{ id, models: [model_id, ...] }] }
   - Logic: Check env vars via CloudEngine, return empty array jika tidak ada key

2. GET /v1/providers/status
   - Return status semua provider
   - Response: { providers: [{ id, name, configured: bool, model_count: int }] }
   - Check engine untuk setiap provider

3. POST /v1/providers/configure
   - Body: { provider_id, api_key, base_url (optional) }
   - Save ke ~/.freya/cloud-keys.env
   - Trigger engine reload
   - Return: { success: bool, message: string }

4. POST /v1/providers/test
   - Test connection ke provider
   - Return: { success: bool, models: [model_ids], error: string|null }

### TAHAP 2: BACKEND - CLOUD ENGINE REFACTOR ✅
--------------------------------------------------------------------------------
FILE: `src/freya/engine/cloud.py`

PERUBAHAN:

1. Hapus hardcoded model lists:
   - _OPENAI_MODELS (REMOVED)
   - _ANTHROPIC_MODELS (REMOVED)
   - _GOOGLE_MODELS (REMOVED)
   - _MINIMAX_MODELS (REMOVED)
   - _CODEX_MODELS (REMOVED)

2. Ganti dengan PROVIDER_META:
   - PROVIDER_META dict dengan metadata provider (nama, prefix model)

3. Fungsi baru di CloudEngine:

   list_available_models() -> Dict[str, List[str]]
   - Return only models jika API key ada
   - Group by provider ID

   get_provider_status() -> List[Dict]
   - Return status semua provider dengan configured flag

   test_provider(provider_id, api_key, base_url) -> List[str]
   - Test connection ke specific provider
   - Return list model IDs jika successful

4. list_models() tetap ada untuk backwards compatibility
   - Return potential models dari semua provider yang configured
